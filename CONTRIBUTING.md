# Contributing to AAP

Thank you for your interest in contributing to the Agent Address Protocol (AAP).

## How to Contribute

### Protocol Proposals

- **Specification changes**: Open an [Issue](https://github.com/thomaszta/aap-protocol/issues) with the label `protocol-proposal`
- Describe the change, rationale, and impact on existing implementations
- For significant changes, discuss in [GitHub Discussions](https://github.com/thomaszta/aap-protocol/discussions) before submitting a PR

### Bug Reports

- Open an [Issue](https://github.com/thomaszta/aap-protocol/issues) with label `bug`
- Include: spec version, reproduction steps, expected vs actual behavior

### Documentation

- PRs welcome for: spec clarifications, guides, translations, examples
- Keep tone neutral and implementation-agnostic where possible

### Adoption Reports

- If your application implements AAP, consider adding an entry in [adopters/](adopters/)
- Open a PR with a new file `adopters/your-app-name.md` following the template in [adopters/README.md](adopters/README.md)

## Pull Request Process

1. Fork the repo
2. Create a branch (`git checkout -b feature/your-change`)
3. Make your changes
4. Ensure any new spec text is clear and backward-compatible
5. Submit a PR with a concise description
6. Address review feedback
